    Arnold (Arnold v0.6.2)

    Arnold main API functions, which can be used to communicate with other nodes or applications. The REST Api utilizes these functions mainly.

    Functions

    analyse(node_id, sensor_id, tag, prediction)

    Returns the correlation with each metric that was pushed for a node. If found positive or negative correlation for a threshold larger (or smaller) than 0.7 marks it as the metrics are correlating with each other.

    feed(node_id, sensor_id, time, value)

    Feed data to the sensors. It is going to be marked for train if the threshold is reached for the current input category.

    predict(id, tag, horizon)

    Predicts a value for a given id and tag. Check out the :windows to see the available tag values. There are two types of predictions: simple and complex. Complex uses the trained neural network, while the simple is only used when there aren't any networks for the current id. It uses SES algorithm for predictions.

    predict(node_id, sensor_id, tag, horizon)

    Predicts a value for a given node, sensor_id and tag. Check out the :windows to see the available tag values. Calls for predict/2.

    predict(id, tag, horizon)

    (since 0.5.4) 
    @spec predict(id :: binary(), tag :: Arnold.Sensor.tag(), horizon :: pos_integer()) ::
  {:ok, Arnold.NeuralNetwork.prediction()}

    Predicts a value for a given id and tag. Check out the :windows to see the available tag values. There are two types of predictions: simple and complex. Complex uses the trained neural network, while the simple is only used when there aren't any networks for the current id. It uses SES algorithm for predictions.
