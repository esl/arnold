    API Reference Arnold v0.6.2

    modules

     Modules

    Arnold

    Arnold main API functions, which can be used to communicate with other nodes or applications. The REST Api utilizes these functions mainly.

    Arnold.Config

    Configuration handler module for Arnold.

    Arnold.Database

    API wrapper for Memento to store or fetch data from the database.

    Arnold.Database.Table.Manager

    Memento table for saving the manager state.

    Arnold.Database.Table.NetworkModel

    Memento table for neural network weights.

    Arnold.Database.Table.Sensor

    Memento table for sensors.

    Arnold.Manager

    The manager responsible for saving sensors state, finished sensor tags and the number of sensor agents. If an agent terminates it the duty of the module to start a new one.

    Arnold.NeuralNetwork

    The neural network API that contains function for training and prediction.

    Arnold.Plug.InvalidGetParams

    Error raised when a required field is missing.

    Arnold.Plug.InvalidPostParams

    Error raised when a required field is missing.

    Arnold.Plug.Router

    Router for the REST API. There are 2 main routes, get and post paths. The body must be a json in all cases.

    Arnold.Sensor

    Contains the API functions for the sensors

    Arnold.Sensor.Supervisor

    Dynamic supervisor which can start Sensor Agents when needed.

    Arnold.Statistics.Decomposition

    Decompose the input values into trend and seasonality.

    Arnold.Statistics.ExponentialSmoothing

    Collection of exponential smoothing algorithms.

    Arnold.Statistics.Math

    Collection of commonly used mathematical/statistic functions.

    Arnold.Supervisor

    Top-level supervisor of the Arnold application.

    Arnold.Utilities

    Collection of utility functions including id generation, truncate and normalization functions.

    Arnold.Utilities.Benchmark

    Collection of benchmark functions
